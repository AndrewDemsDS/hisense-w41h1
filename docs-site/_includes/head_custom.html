{%- comment -%} Dark mode, applied BEFORE first paint. The theme's own jtd.setTheme() only becomes available once just-the-docs.js has run at the end of the body, so using it alone would render the light stylesheet first and then repaint dark: a white flash on every navigation, which is worse than having no dark mode. This runs in instead and rewrites the stylesheet href directly, so the correct scheme is the only one ever fetched. just-the-docs has no native prefers-color-scheme support: its docs offer only a static `color_scheme` setting plus jtd.setTheme(), so automatic switching has to be added here. Two things were checked against the deployed theme rather than assumed: the scheme stylesheet is the FIRST in the document (which is also what jtd.setTheme itself assumes), and just-the-docs-dark.css is served alongside the default. If a future theme version reorders those, the guard below leaves the page on its default scheme rather than breaking styling entirely. Preference order: an explicit choice the reader made (localStorage), otherwise the operating system's setting. Readers who never touch the toggle simply get what their OS asked for. {%- endcomment -%}
